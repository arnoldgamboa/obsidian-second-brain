<span style="font-size:11.142857551574707pt;">The following aspects need to be considered as we plan for this project:</span>

<span style="font-size:11.142857551574707pt;"><b>The problem</b></span>

<span style="font-size:11.142857551574707pt;">The most successful web applications are born out of the desire to solve a particular problem. As we start working on this project, we have to identify the existing problem we’re trying to solve — how can we make their lives simpler, easier and more productive? How can we potentially make their operations more efficient and effective?</span>

<span style="font-size:11.142857551574707pt;">All these has to be determined first before we start. This way, we create</span> <span style="font-size:11.142857551574707pt;"><b><i>value</i></b></span><span style="font-size:11.142857551574707pt;">, even creating a need that they don’t even perhaps know.</span>

<span style="font-size:11.142857551574707pt;"><b>The target audience</b></span>

<span style="font-size:11.142857551574707pt;">We determine our target audience — what’s their current process, what makes their process inefficient, what makes their work harder to do and how we can potentially make their lives easier. In programming, this is what we call “systems analysis and design”. We have to know the existing process, gather facts, list problems, know the users of the existing process and design a system for them.</span>

<span style="font-size:11.142857551574707pt;">This is where face to face meeting and interviews with actual potential users of the system will be very beneficial. By interviewing the potential users, we can determine first hand their “pain points” and therefore become more prepared to suggest solutions.</span>

<span style="font-size:11.142857551574707pt;"><b>The input/output process</b></span>

<span style="font-size:11.142857551574707pt;">This is where we will need to collect vital information and documents. We need to know the data involved, the required reports and the process that goes between the current input of data to creation of relevant reports.</span>

<span style="font-size:11.142857551574707pt;">We need to ask questions like: is the process manually done? If yes, how can we automate it. If no, how can we improve it?</span>
