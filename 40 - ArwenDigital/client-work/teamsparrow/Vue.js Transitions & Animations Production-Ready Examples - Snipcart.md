<span style="font-size:17.14285659790039pt;">Vue.js "transformations"—context</span>
 **(unknown attachment: public.jpeg)** 

<span style="font-size:11.142857551574707pt;"><i>Disclaimer: this post digs into Vue.js and the tools it offers with its transition system. It is assumed that you are already comfortable with the basics of Vue.js and CSS transitions. For the sake of brevity and clarity, we won't get into the "logic" used in the demo.</i></span>
<span style="font-size:11.142857551574707pt;"><b>Animations & transitions</b></span><span style="font-size:11.142857551574707pt;">. They're the final layer, the proverbial cherry on top, that brings your website or web-app to life and entices users to explore. Animations and transitions are an integral part of UX & UI design. They are, however, so very easy to get wrong. In complex situations like dealing with lists, they can be nearly impossible to reason about when relying on native JavaScript and CSS. Whenever I ask backend developers why they dislike the frontend so vehemently, their response is usually somewhere along the lines of "</span><span style="font-size:11.142857551574707pt;"><i>... animations</i></span><span style="font-size:11.142857551574707pt;">".</span>
<span style="font-size:11.142857551574707pt;">Even for those of us who are drawn to the field by an urge to create intricate micro-interactions and smooth page transitions, it quickly becomes difficult to reason about our work, given the need to rely on CSS for performance purposes.</span>
<span style="font-size:11.142857551574707pt;">This is where frameworks like Vue.js step in, taking the guess-work and clumsy chains of</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">setTimeout</span> <span style="font-size:11.142857551574707pt;">functions out of transitions. There are already countless articles out there that outline basic usage patterns, but few that show you how to use them together in the real world. In this post, I will dive into implementations of Vue's transition and animation tools that could easily be used in your future projects.</span>
<span style="font-size:17.14285659790039pt;">Transitions vs. animations: key differences</span>
 **(unknown attachment: public.jpeg)** 

<span style="font-size:11.142857551574707pt;">Before we get started, I'd like to clear something up.</span>
<span style="font-size:11.142857551574707pt;">The terms transitions and animations are often used interchangeably but are not, in my experience, one and the same:</span>
- <span style="font-size:11.142857551574707pt;">A</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">transition</span> <span style="font-size:11.142857551574707pt;">is a simple change in the style properties on an element to be transitioned in a single step. They are often handled purely through CSS.</span> 
- <span style="font-size:11.142857551574707pt;">An</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">animation</span> <span style="font-size:11.142857551574707pt;">is a more complex, usually multi-step and sometimes continuous change in appearance. Animations will often call on JavaScript to pick up where CSS' lack of logic drops off. It could be to add a class and trigger the animation or to make complex state transitions that reflect onto the DOM.</span> 
<span style="font-size:11.142857551574707pt;">It is an important distinction when stepping into the world of Vue because both have very different approaches and toolboxes.</span>
<span style="font-size:17.14285659790039pt;">Using Vue.js animations & transitions in production</span>

<p style="text-align:center;margin:0"><span style="font-size:11.142857551574707pt;">A rough overview of Vue transformations on</span> <a href="https://spektrummedia.com/en/" rel="noopener" class="external-link" target="_blank" style="font-size:11.142857551574707pt;color:#e4afaff;">Spektrum's site</a>

<span style="font-size:16.045713424682617pt;">1. Transitions</span>

<span style="font-size:11.142857551574707pt;">The simplest way to achieve transition effects on your page is through Vue's component. It makes things so simple, it almost feels like cheating. Vue will detect if any CSS animations or transitions are being used and will automatically toggle classes on the transitioned content, allowing for a perfectly timed transition system and complete control.</span>
<span style="font-size:11.142857551574707pt;">First step is to identify our scope. We tell Vue to prepend the transition classes with</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">modal</span><span style="font-size:11.142857551574707pt;">, for example, by setting the component's</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">name</span> <span style="font-size:11.142857551574707pt;">attribute. Then to trigger a transition all you need to do is toggle the content's visibility using the</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">v-if</span> <span style="font-size:11.142857551574707pt;">or</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">v-show</span> <span style="font-size:11.142857551574707pt;">attributes. Vue will add/remove the classes accordingly.</span>
<span style="font-size:11.142857551574707pt;">There are two "directions,"</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">enter</span> <span style="font-size:11.142857551574707pt;">and</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">leave</span> <span style="font-size:11.142857551574707pt;">transitions. The former is used when the element is going from hidden to visible and, naturally, the latter is for the other way around.</span>
<span style="font-size:11.142857551574707pt;">Vue then provides 3 "hooks" that represent different timeframes in the transition:</span>
- <span style="font-family:Courier;font-size:13.714285850524902pt;">.tray-enter-active / .tray-leave-active</span><span style="font-size:11.142857551574707pt;">: These will be present throughout the entire transition and should be used to apply your CSS</span> <span style="font-family:Courier;font-size:13.714285850524902pt;">transition</span> <span style="font-size:11.142857551574707pt;">declaration. You can also declare styles that need to be applied from beginning to end.</span> 
- <span style="font-family:Courier;font-size:13.714285850524902pt;">.tray-enter / .tray-leave</span><span style="font-size:11.142857551574707pt;">: Use these classes to define how your element looks before it starts the transition.</span> 
- <span style="font-family:Courier;font-size:13.714285850524902pt;">.tray-enter-to / .tray-leave-to</span><span style="font-size:11.142857551574707pt;">: You've probably already guessed, these determine the styles you wish to transition towards, the "complete" state.</span>
