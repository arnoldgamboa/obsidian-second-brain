Project name: AIMS (Aloysian Information Management System)
AIMS  contains personal data of all SMS alumni at the time they graduated from the school, annual dues and donations information, and basic reporting stuff. Some alumni managed to have their personal data updated through the help of some admin users such as ASMSI officers and staff. 


Analysis of current web application
- it is written with .NET technology/tools
- has alumni database
- donations summary
- admin settings/login

Two ways to approach this project:

1. Maintain the original one and then extend it with a separate app with new features
	1. Advantage - cost effective as you don’t have to pay for redevelopment
	2. Disadvantage - 
		1. integration and continuity of the project is limited.
		2. maintaining 2 system is more burdensome
1. Redevelop the main app from the ground up and then create the new features
	1. Advantages
		1. Easier to maintain in the long run. You’d be maintaining only one web app and code base
		2. Extendible for future updates
		3. Continuity
	1. Disadvantage
		1. Cost for redevelopment

**Approach Options**
		**Maintain the original app and extend it with a separate app**
		- **Advantages**
		- Cost-Effective: Less expenditure is required as there is no need for redevelopment.
		- **Disadvantages**
		- Limited Integration: Due to separate apps, the integration and continuity are restricted.
		- Maintenance Burden: Operating two systems may become cumbersome over time.
		**Redevelop the main app and extend features**
		- **Advantages**
		- Maintenance Ease: One unified web app simplifies long-term maintenance.
		- Future-Proof: Easier to extend and update in the future.
		- Continuity: Ensures a seamless user experience.
		- **Disadvantages**
		- Higher Initial Cost: Redevelopment will be more expensive upfront.
