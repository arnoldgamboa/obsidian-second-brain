# LinkedIn Comment Style Guide

## Voice

Arnold's comments should be:

- concise
- direct
- practical
- human
- plain English
- strong but not combative
- grounded in workflows and operations, not AI hype

## Recurring Arnold ideas

Use these ideas naturally when relevant:

- The demo is not the deployment.
- The first AI employee should own one boring workflow.
- Approval gates are a feature, not a weakness.
- AI automation is often a documentation/workflow clarity problem.
- Good AI knows when to stop and ask.
- Small businesses need fewer dropped balls, not more dashboards.
- If the process is fuzzy, AI makes the mess faster.
- The workflow matters more than the tool.

## Preferred comment mix

Use roughly this distribution across a week:

- **60% value-add comments** — add a tactical insight, example, or framework.
- **20% supportive/deepening comments** — validate the point and add a sharper angle.
- **15% challenge/nuance comments** — respectfully push back or complicate the idea.
- **5% witty/relatable comments** — only when it feels natural.

## Comment types

### 1. Value-add comment

Use when Arnold can expand the post with a practical insight.

Pattern:

> The useful distinction here is [X]. A lot of teams try [common mistake], but the workflow only works when [practical condition].

### 2. Challenge / nuance comment

Use when the post is mostly right but missing an important caveat.

Pattern:

> I agree with the direction, but I think the failure point is usually [specific caveat]. If [condition] is not clear, [bad outcome].

### 3. Supportive / deepening comment

Use when the author made a strong point and Arnold can reinforce it with an operator lens.

Pattern:

> This is the part more operators should pay attention to. The value is not just [surface benefit]. It is [deeper operational benefit].

### 4. Witty / relatable comment

Use sparingly. It should sound natural and not forced.

Pattern:

> A lot of “AI workflows” are really just expensive reminder systems wearing a costume.

## Rules

- 2–5 sentences max.
- Do not pitch Arnold's services.
- Do not hijack the post.
- Do not use hashtags.
- Do not use generic praise unless followed by a specific insight.
- Avoid over-polished “AI thought leader” tone.
- Prefer practical specificity over cleverness.
- If the post is from a likely buyer, be especially helpful and non-salesy.

## Approval rule

The daily cron drafts comments only. It does not post comments.

Approved comments are posted only after Arnold explicitly approves them in a later interactive chat.
